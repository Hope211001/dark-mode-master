# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm run dev         # Vite dev server on host :: port 8080
npm run build       # Production build (vite build)
npm run build:dev   # Development-mode build (sourcemaps, no minify)
npm run lint        # ESLint flat config (eslint.config.js)
npm run preview     # Serve the production build locally
```

There is no test runner configured. Path alias `@` resolves to `./src` (see [vite.config.ts](vite.config.ts)).

Required env vars (read via `import.meta.env`, must be prefixed `VITE_`):
- `VITE_API_URL` — base URL for the backend, e.g. `http://localhost:5000/api`. **Note the inconsistency:** [src/services/client.tsx:5](src/services/client.tsx#L5) appends `/api`, but [src/components/client/ClientHeader.tsx:21](src/components/client/ClientHeader.tsx#L21) and [src/components/client/ClientSidebar.tsx:27](src/components/client/ClientSidebar.tsx#L27) reuse the same var as the **Socket.IO origin** (no `/api`). Local dev with `VITE_API_URL=http://localhost:5000/api` works because Socket.IO ignores the path; if you ever set the var to a path-less origin, the axios calls will 404. Don't "fix" one without the other.
- `VITE_GOOGLE_CLIENT_ID` — required at boot in [src/main.tsx](src/main.tsx); app refuses to render without it
- `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY` — used by [src/lib/supabase.tsx](src/lib/supabase.tsx) (anon key only — never the service role)

## Architecture

Vite + React 18 + TypeScript + shadcn/ui (Radix primitives in [src/components/ui/](src/components/ui/)) + Tailwind + React Router 6. Server state via TanStack Query, forms via react-hook-form + zod. The backend lives in `../Back` (Express + Supabase + Stripe + Socket.IO).

**Two role-gated portals on a single SPA.** [src/App.tsx](src/App.tsx) declares public routes plus two nested route trees: `/admin/*` (role `admin`) and `/client/*` (role `client`). Every protected child is wrapped individually in `<ProtectedRoute allowedRoles={[...]}>` — the wrapper is **not** applied at the parent `<Route>` level. When adding a new portal route, replicate the per-leaf wrapping or it will be public.

**Auth flow.** [src/contexts/AuthContext.tsx](src/contexts/AuthContext.tsx) holds the `user`/`loading`/`isAuthenticated` state. On mount it calls `authService.getProfile()` if a token exists in `localStorage`; on failure the token is purged. **All routes must wait on `loading`** — [src/components/ProtectedRoute.tsx](src/components/ProtectedRoute.tsx) shows a spinner while loading is true, otherwise a logged-in user is briefly seen as logged-out and redirected to `/login`. JWT is stored in `localStorage` under key `token` and attached as `Authorization: Bearer ...` by the axios interceptor in [src/services/client.tsx](src/services/client.tsx). On 401 the interceptor wipes the token and hard-redirects to `/login`.

**API layer.** All HTTP goes through the singleton `apiClient` in [src/services/client.tsx](src/services/client.tsx). Each domain has a sibling `*.service.tsx` (auth, leads, zones, notifications, stripe, invoices, cookies, subscription, user, config) — keep new endpoints inside these service files, never call `axios` directly from components.

**"Notifications" = leads.** [src/services/notification.service.tsx](src/services/notification.service.tsx) hits `/notifications` which the backend serves from the `leads` table. The unread badge in [src/components/client/ClientHeader.tsx](src/components/client/ClientHeader.tsx) and [src/components/client/ClientSidebar.tsx](src/components/client/ClientSidebar.tsx) opens its own Socket.IO connection (`new_lead_notification` event) — there are **two independent socket clients** for the same stream. If you add another live consumer, factor a shared client rather than opening a third connection.

**Socket.IO requires the long-running backend.** The backend's Vercel deployment does not run Socket.IO (see `../Back/CLAUDE.md`). Realtime features must degrade gracefully — the notifications panel already polls via `getAll()` on mount, so a missing socket only loses live push.

**shadcn/ui conventions.** Components were generated by the shadcn CLI ([components.json](components.json), style `default`, alias `@/components/ui`). Prefer composing existing primitives in [src/components/ui/](src/components/ui/) over adding new dependencies. Tailwind config lives in [tailwind.config.ts](tailwind.config.ts).

**Lovable tagger.** [vite.config.ts](vite.config.ts) injects `lovable-tagger` only in `mode === "development"` — it annotates JSX for the Lovable.dev visual editor and must not ship to production. Don't enable it in `build` mode.

**Stale files to ignore.** [src/backup/](src/backup/), [src/pages/Index copy.tsx](src/pages/Index%20copy.tsx) and [src/pages/Index copy 2.tsx](src/pages/Index%20copy%202.tsx) are dead snapshots. The live homepage is [src/pages/Index.tsx](src/pages/Index.tsx).
